### 🧪 元素附着机制详解

```
核心规则:
1. 每次攻击会为目标附加对应元素的"附着层数"(1~3 层)
2. 当两种不同元素同时存在于同一目标时 → 自动触发反应
3. 反应后消耗各 1 层附着，剩余层数可重新积累

战斗流程示例:
┌─────────────────────────────────────────────┐
│ 步骤 1: 战士使用火球术                      │
│   ↓                                        │
│   敌人附加 Fire(2 层)                        │
│                                           │
│ 步骤 2: 法师使用水枪                       │
│   ↓                                        │
│   检测到双元素共存 (Fire+Water)            │
│   → 触发蒸发反应 ×2.0x                     │
│   → Fire 和 Water 各消耗 1 层               │
│                                           │
│ 步骤 3: 剩余附着                           │
│   Fire(1), Water(1)                       │
│   → 再次攻击可继续触发反应!                 │
└─────────────────────────────────────────────┘
```

## 🎯 二、增援范围机制 (核心亮点 - 完整参数)

### 🔬 详细参数表

| 参数名称 | 推荐值 | 可调范围 | 默认值说明 | 战术意义 |
|----------|--------|----------|------------|----------|
| **enforcement_radius** | 64px | 32~128px | 以怪物为中心，半径内其他敌人可被拉入战斗 | 控制引怪风险范围 |
| **alert_range_multiplier** | 1.5x | 1.0~2.0x | ALERTED 状态下增援范围扩大系数 | 警戒时更易触发群殴 |
| **max_enforcement_count** | 3 个 | 1~5 个 | 单次战斗最多激活的增援敌人数量 | 限制爆发上限 |
| **enforcement_delay** | 0.5s | 0.2~1.0s | 从触发到增援加入的战斗延迟 (避免瞬间爆发) | 给玩家反应时间 |

### 📋 警戒状态范围扩大逻辑

```
ALERTED 状态: enforcement_radius *= 1.5x (巡逻怪触发时)
PATROLLING 状态：enforcement_radius *= 1.2x (普通巡逻时)

玩家主动规避策略:
├─ is_stealthed = true → ✅ 可避免触发增援
└─ 距离 > enforcement_radius × 2.0 → ✅ 可在警戒前撤离
```
