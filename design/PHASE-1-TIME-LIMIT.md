# Phase 1: 单局时长控制方案

**时间**: 2026-03-18  
**目的**: 将地牢探索 + 搜打撤单局控制在 **20-30 分钟**  

---

```
┌─────────────────────────────────────────────┐

│              ↓                               │

└─────────────────────────────────────────────┘
```

## ⏱️ 单局时长控制方案（推荐）

### 方案 A：地牢层数限制 ✅ **推荐**

| 参数 | 建议值 | 说明 |
|------|--------|------|
| **最大层数** | 3-5 层 | 每层约 5-7 分钟 |
| **撤离点位置** | 随机生成在第 2-4 层 | 增加策略性 |
| **强制撤离条件** | 生命值<20% / 超时警告 | 防止拖沓 |

#### 优点
- ✅ 明确的目标感（打到哪层就撤）
- ✅ 可控的单局时长
- ✅ 增加决策压力（贪心 vs 保守）

#### 缺点
- ⚠️ 需要精心设计每层难度曲线

---

### 方案 B：资源目标导向

| 参数 | 建议值 | 说明 |
|------|--------|------|
| **任务目标** | 收集 X 个特定资源 | 随机生成任务 |
| **背包容量限制** | 固定格数（如 12 格） | 需要取舍 |
| **撤离倒计时** | 最后 5 分钟提示 | 增加紧张感 |

#### 优点
- ✅ 每局目标不同，重复可玩性高
- ✅ 背包管理增加策略深度

#### 缺点
- ⚠️ 可能出现刷资源导致时长不可控

---

### 方案 C：混合模式（最推荐）🔥

```
单局流程:
1. 进入地牢 → 探索第 1 层 (~5min)
2. 随机任务生成 → "收集 X 个炼金材料" (~8-10min)
3. 撤离点提示出现 → 开始规划撤离路线 (~2-3min)
4. 强制撤离倒计时 → 最后 5 分钟警告 (~5min)

总时长：约 20-30 分钟
```

#### 优点
- ✅ 结合层数限制和目标导向
- ✅ 既有明确目标又有探索自由
- ✅ 撤离倒计时增加紧张感

## 🎮 Phase 1 移动速度参数调整建议

基于**慢节奏 + 20-30 分钟单局**的核心设定：

### 推荐参数

| 参数 | 建议值 | 理由 |
|------|--------|------|
| **最大速度** | **200 px/s (偏慢)** | 符合慢节奏，强调策略而非反应 |
| **加速度** | **600-800 px/s² (较低)** | 有重量感，避免过于敏捷 |
| **减速度/摩擦力** | **500-700 px/s²** | 缓慢停止，增强"重量感" |

### 设计理由

```
慢节奏 = 
  - 更强调策略规划（路线选择、资源管理）
  - 降低操作门槛（不需要快速反应）
  - 增加沉浸感（感受角色的"重量"和移动过程）
  
这符合：
  ✅ 基地建设玩法的基调
  ✅ 搜打撤的紧张但不慌乱的感觉
```

## 📊 单局时长分解参考

| 阶段 | 耗时 | 说明 |
|------|------|------|
| **进入地牢** | ~1min | 加载 + 初始探索 |
| **探索第 1-2 层** | ~8-10min | 熟悉环境、收集基础资源 |
| **完成任务目标** | ~8-10min | 搜刮特定资源/击败 BOSS |
| **规划撤离路线** | ~3-5min | 风险评估、路径选择 |
| **撤离过程** | ~5min | 紧张的最后冲刺 |
| **总计** | **20-30 分钟** | - |

## 💬 Phase 1 移动参数最终建议

基于上述分析，Phase 1 的推荐参数：

```gdscript
# Player Controller (player_controller.gd)
@export var max_speed: float = 200.0      # 最大速度 (偏慢)
@export var acceleration: float = 700.0   # 加速度 (较低，有重量感)
@export var deceleration: float = 600.0   # 减速度 (缓慢停止)

# 手感定位：
# - 不是"敏捷刺客"风格
# - 而是"稳重战士"风格
# - 移动时有明显的加速/减速过程
```
