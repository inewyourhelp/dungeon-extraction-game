## 📋 最终推荐方案

### 🔹 一、单局时长控制：**混合模式** ✅

```
单局流程:
┌─────────────────────────────────────┐
│ 1. 进入地牢 → 探索第 1 层      (~5min) │
│ 2. 随机任务生成 → 收集资源    (~8-10min)|
│ 3. 撤离点提示出现 → 规划路线   (~2-3min)|
│ 4. 强制撤离倒计时 → 最后冲刺     (~5min) │
└─────────────────────────────────────┘

总时长：约 20-30 分钟
```

**关键机制**:
- ✅ **层数限制**: 最大 3-5 层，防止无限探索
- ✅ **随机任务**: "收集 X 个炼金材料"，增加重复可玩性
- ✅ **撤离倒计时**: 最后 10 分钟提示 + 最后 5 分钟警告

---

### 🔹 二、Phase 1 移动参数：**慢节奏重量感** ✅

基于"稳重战士"风格定位：

```gdscript
# Player Controller (player_controller.gd)
@export var max_speed: float = 200.0       # 最大速度（偏慢）
@export var acceleration: float = 700.0    # 加速度（较低，有重量感）
@export var deceleration: float = 600.0    # 减速度（缓慢停止）

# 手感定位：
# - 不是"敏捷刺客"风格 ← 快速反应、瞬间启动
# - 而是"稳重战士"风格 ✅ 
#   - 移动时有明显的加速/减速过程
#   - 强调路线规划而非操作精度
```

**设计理由**:
- ✅ 符合基地建设 + 搜打撤的紧张但不慌乱的感觉
- ✅ 降低操作门槛，适合策略型玩家
- ✅ 增加沉浸感（感受角色的"重量"和移动过程）

---

### 🔹 三、撤离机制：**随机位置** ✅

```
撤离点设计:
- [x] **随机生成在第 2-4 层**: 增加策略性
- [ ] 固定位置撤离点：过于简单
- [ ] 需要钥匙解锁：增加复杂度（Phase 2+）

撤离警告机制:
- 最后 10 分钟：屏幕边缘闪烁提示
- 最后 5 分钟：倒计时显示 + 音效提醒
```

**设计理由**:
- ✅ 每次撤离路线都不同，增加重复可玩性
- ✅ 需要评估风险（距离 vs 敌人密度）
- ✅ 与"搜打撤"的核心体验高度契合
