## 🔄 **使用 gh CLI 的日常工作流程**

### 📥 **开始工作前** (每次任务必做!)

```bash
# 方式 A: gh cli (推荐!优先使用!)
gh pr list --base zhongshu-dev --state open | Select-String "Merge" # 查看是否有未合并的 PR

# 方式 B: git 拉取最新代码
gh repo clone inewyourhelp/dungeon-extraction-game --depth 1 --no-tags 2>/dev/null || true
git pull origin feature && git pull zhongshu-dev zhongshu-dev

# 检查工作状态
gh pr status
git status
```

### 💻 **工作完成后**

```bash
# 1. 提交代码 (规范提交信息)
git add .
git commit -m "feat: 新增玩法设计 #1"

# 2. 推送到自己分支
git push origin zhongshu-dev

# 3. 创建 Pull Request 合并到 feature 分支
#    使用 gh pr create (GitHub CLI 优先!推荐!)
gh pr create \
  --base feature \
  --head zhongshu-dev:zhongshu-dev \
  --title "feat: 新增策划内容" \
  --body "描述本次策划工作...\n\n变更：\n- [列表内容]\n\n测试状态：待审核"

#    或者使用 git merge (备选方案)
git pull origin feature && git merge zhongshu-dev -m "Merge zhongshu-dev into feature"
```

## 📋 **提交规范** (Conventional Commits)

- `feat:` 新功能 (feature) ← 策划新增内容
- `fix:` 修复 bug
- `doc:` 文档更新
- `style:` 代码格式调整
- `refactor:` 重构代码
- `perf:` 性能优化
- `chore:` 构建/工具链修改
- `test:` 测试用例新增

**示例：**
```bash
git commit -m "feat: 新增地下城主角色设定 #1"
git commit -m "fix: 修正迷宫关卡数值平衡"
git commit -m "doc: 完善美术需求文档描述"
```
